# Security Policy

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability related to Excalibur, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### Contact

- **Email:** contact@lorenzofradeani.com
- **Response time:** Within 72 hours for initial acknowledgment
- **Resolution target:** Critical vulnerabilities addressed within 14 days

### Scope

This security policy covers:
- The Excalibur framework and its distributed components
- The licensing and access control process
- This public repository and associated communications

### Out of Scope

- Vulnerabilities in third-party dependencies (report to the upstream maintainer)
- Social engineering attacks against Excalibur personnel
- Denial of service attacks
- Issues already known and documented

### Disclosure

We follow coordinated disclosure. We will:
1. Acknowledge your report within 72 hours
2. Provide an estimated timeline for a fix
3. Notify you when the fix is deployed
4. Credit you in the fix announcement (if you wish)

We ask that you:
1. Do not publicly disclose the vulnerability before it is fixed
2. Do not access or modify data belonging to other users
3. Act in good faith to avoid disruption to services

## Supported Versions

Only the latest release of Excalibur is supported with security updates. Licensed customers receive security patches through their existing distribution channel.
